## 1. 한 문장 요약

**방문자는 태양계를 인터랙티브하게 탐험하면서, 시각화와 실제 사이의 거리를 깨닫는다.**

---

## 2. 동기 (Why)

어릴 적, 칼 세이건의 *Pale Blue Dot* 사진을 처음 봤을 때 — 보이저 1호가 60억 km 밖에서 찍은 지구가 단 한 픽셀의 푸른 점이었던 그 사진. 그 이미지는 *우주에서 우리가 얼마나 작은가* 를 가르치기보다, *우주가 얼마나 큰가* 를 가르쳤다. Phase 2는 그 깨달음을 코드로 다시 쓰는 일이다.

우리가 책에서, 다큐에서, 심지어 NASA 공식 일러스트에서 보는 태양계는 모두 *거짓말*이다 — 정확하게는, *교육적 거짓말*. 행성들이 그렇게 가까이 모여 있다면 우리는 안드로메다보다 빠른 충돌을 매일 봤을 것이다. 진짜 태양계는 *대부분 빈 공간*이다. *그리고* 모든 행성이 비슷한 속도로 자전하는 다큐의 모습도 거짓말이다 — 수성은 한 바퀴 도는데 두 달, 금성은 여덟 달, 목성은 열 시간. 케플러 제3법칙이 만드는 *비선형 비례* 까지 두 차원 모두 거짓말로 표현된다.

Phase 2는 그 *두 거짓말* 을 보여주고, 두 진실로 토글하는 무대다.

기술적으로는 — Three.js + React Three Fiber로 첫 본격 3D 작업, 디자인 토큰 시스템(Phase 1)이 셰이더까지 흘러가는지 검증, 그리고 *"수치를 그대로 쓰면 보이지 않는다"* 라는 시각화의 본질적 트레이드오프를 의식적으로 다루는 첫 무대.

## 3. 사용자 시나리오

**60초 시나리오**:

1. **(0~10s)** 페이지 로드. 어두운 우주, 중앙에 태양. 8개의 행성이 자기 궤도를 도는 *시각적 모드*가 기본 — 거리도 자전도 *압축된 비례* 라 모든 행성이 한 화면에서 비슷한 리듬으로 살아 움직인다. 한눈에 *"아, 태양계"* 가 들어온다. (sub-2-4 갱신: 페이지 진입 시 *항상 visual 모드 + 1× 속도* 로 자동 리셋 — F5 든 HMR 이든 동일.)
2. **(10~25s)** 행성에 호버하면 이름 + 기본 정보. 클릭하면 카메라가 그 행성으로 줌인. 지구를 클릭하면 달까지 같이 보인다.
3. **(25~45s)** 화면 하단 컨트롤 패널: **[자전 토글]** **[거리 토글]** **[전체 진실 프리셋]** **[시간 속도]** **[정지 ⏸ / 리셋 ↺]** **[카메라 리셋]**.
   - *자전 토글* 을 누르는 순간 — 수성과 금성이 *거의 정지* (visual 81h → real 1407.6h, 17배 느려짐), 목성이 *흐릿한 가속* (visual 18.4h → real 9.93h, 1.85배 빨라짐) 으로 변한다. 자전 비례가 17배 차이라는 *시간의 진실*.
   - *거리 토글* 을 누르는 순간 — 행성들이 화면 밖으로 흩어지고, 지구가 점이 되어 사라진다. 해왕성은 22배 멀어져 *완전히 화면 밖* — 공간의 진실. (수성은 태양 *밖에서* 작은 점으로 보임 — `distanceScale` 도 함께 보간되어 *물리적 충돌 회피*.)
   - *전체 진실 프리셋* 을 누르면 — 두 변환이 동시에 1.5초간 일어난다. *교육적 거짓말이 진실로 동시에 무너지는 순간*. 양쪽 `changedAt` 동일 timestamp 로 *정확한 동기성* 보장.
4. **(45~60s)** *"어..."* 의 순간. 시간 정지로 멈추고 둘러보거나, 시간 속도를 최대로 올려 해왕성이 한 바퀴 도는 데 *얼마나 오래*인지 체감한다. 길을 잃으면 **리셋 [↺] 한 번이면 처음으로** — *시간도 모드도* 모두 visual + 1× 깨끗한 시작점. (sub-2-4 갱신: R 버튼 / ↺ 클릭 / 페이지 재진입이 *모두 동일 효과*. *한 액션 = 한 의도*.)

**기억에 남길 두 순간** (둘이 합쳐 PRD의 영혼):
- *자전 토글을 누르는 순간 수성이 멈추고 목성이 광속이 되는 1초* — 시간의 진실
- *거리 토글을 누르는 순간 지구가 점이 되는 1초* — 공간의 진실

두 토글은 *교육적 거짓말 → 우주의 진짜 모습* 변환의 두 차원. 한 차원만 있어도 임팩트는 살지만, 두 차원 모두 살리면 *우리가 다큐에서 본 태양계가 얼마나 거짓이었나* 가 두 배로 분명해진다.

*sub-2-4 메모*: *크기* 의 거짓말은 *유지* 한다 — radius 토글은 PRD §4 범위 외. 행성을 *볼 수 있게* 유지하는 것이 *거리/시간의 진실을 전달* 하는 전제. 세 차원 중 두 차원만 진실로 토글한다.

## 4. 범위

### ✅ 포함 (6개)

1. **8개 행성 + 태양 3D 시각화 + 고리 (토성, 천왕성)** — 저폴리곤 sphere + NASA 텍스처, 토성은 고리 텍스처 / 천왕성은 단색 회색 띠
2. **달** — 지구의 위성으로 함께 표시 (*"우리 위성"* — 거리 토글 시 또 하나의 깨달음 포인트)
3. **진실 토글 (거리 + 자전 각각) + 프리셋 버튼**
   - **거리 토글**: 행성 *거리* (위치 + 궤도 라인) 를 비례 압축 ↔ 실제 비율로 보간.
     - visual: `distanceExponent=0.5` + `distanceScale=20` → 행성들 한 화면 모임
     - real:   `distanceExponent=1.0` + `distanceScale=80` → 비례 그대로 + 광활한 거리, *지구가 점이 되는 1초*
     - *sub-2-4 [Fix]*: `distanceScale` 도 보간 대상에 포함 — 수성이 태양 안으로 박히지 않게 + 해왕성이 *진짜* 화면 밖으로
   - **자전 토글**: 자전 *주기* 를 압축 ↔ NASA 원본으로 보간.
     - visual: `getVisualRotationPeriod` = `sign(p) × (|p|/24)^0.3 × 24` (24h 기준점 보존 + sqrt 보다 강한 압축)
     - real:   NASA 원본값 그대로
     - *sub-2-4 [Fix]*: exponent 0.5 → 0.3 — *수성 멈춤 / 목성 광속* 의 시각적 임팩트가 *체감 가능* 수준 (수성 17배 차이)
     - 부호 보존: 금성/천왕성 역회전이 visual 모드에서도 유지
     - 축 기울기 보정 (`getEffectiveRotationPeriod`) 은 보간 *후* 호출 측에서 합성
   - **전체 진실 프리셋**: 두 토글을 *동시에* 진실 모드로. 1.5초 보간 동기화. 양쪽 모두 'real' 이면 다음 누름은 *둘 다 visual* 로 (탈출 경로).
   - 공전 속도는 자전과 무관하게 시간 슬라이더로 제어 (케플러 비례는 항상 살아있음)
4. **시간 컨트롤** — 버튼 그룹 (정지 / 0.1× / 1× / 100× / 10,000×) + 키보드 단축키 (Space, 1-4, R) + **통합 리셋 버튼** ([↺] / R 키 → 시간 + 모드 모두 처음)
5. **행성 호버/클릭 인터랙션** — 호버 시 정보, 클릭 시 카메라 줌인 + 카메라 리셋 버튼 (sub-2-5)
6. **단순 starfield 배경** — 별자리 라인 없음 (perspective 문제)

### ❌ 제외

- **별자리 라인** → perspective 문제 (지구 시점에 묶이는 데 카메라가 자유로움)
- **달 외 위성들** (목성 갈릴레이 위성 등) → 추후 Phase 또는 미니 프로젝트
- **소행성대 / 카이퍼 벨트** → 시각적 노이즈, 메시지 흐림
- **행성 표면 상세** (분화구, 폭풍 등) → 텍스처로 충분, 절차적 디테일은 과잉
- **외계행성** → Phase 6
- **중력 시뮬레이션** (행성이 서로 끌어당기는) → Phase 3 N-body
- **크기 (radius) 토글** → 행성이 *보여야* 거리/시간의 진실이 전달됨. 세 차원 중 *크기는 시각 모드 유지*.
- **시간만 리셋 / 모드만 리셋 분리** → sub-2-4 시도 후 통합. *한 사용자 의도 = 한 액션* (YAGNI). 분리 필요해지면 그때 별도 액션.

## 5. 성공 기준

### 정성적
- *"자전 토글 순간"* 과 *"거리 토글 순간"* 모두 충격을 준다 (자체 테스트: 처음 본 친구 3명이 둘 다에서 *"어..."* 라고 말하는가)
- *"전체 진실 프리셋"* 이 두 변환의 *동기화된 임팩트* 를 만든다 (한 토글의 임팩트를 단순 합산한 것 이상의 효과)
- 시각화와 실제의 거리감이 *수치*가 아니라 *체감*으로 전달된다 — 거리, 시간 두 차원에서
- 모바일에서 만져봐도 답답하지 않다
- 실수해도 괜찮은 컨트롤 — **리셋 한 번이면 처음으로**. 시간 / 진실 모드 / 어떤 상태에서든 *↺ 한 클릭 = visual + 1×*.

### 정량적
- 데스크톱 **60fps 유지** (8개 행성 + 달 + starfield + 카메라 컨트롤 + 토글 보간 중)
- 모바일 **30fps 이상**
- **Lighthouse Performance 85+** (3D는 90+ 어려움)
- **첫 인터랙티브까지 < 3s** (3D 자산 로드 포함)
- **토글 보간 1.5초** ± 50ms (자전/거리 양쪽 동일, 프리셋 시 두 보간 정확히 동기)
- **수성 안 박힘 보장**: real 모드 도달 시점에 *수성 거리 > 태양 반지름* (sub-2-4 [Fix] 정량 기준 — 자동 테스트로 보장)
- **자전 차이 인지 가능**: visual ↔ real 의 자전 속도 비율이 *수성 기준 17배 이상* (sub-2-4 [Fix])

## 6. 비기능 요구사항

- **성능**: WebGL2 기반, 60fps 데스크톱 / 30fps 모바일. 지오메트리 최소화 — 행성 = 저폴리곤 sphere + 텍스처
- **접근성**: 토글/컨트롤은 키보드로도 조작 가능, 색맹 대비 (행성 식별을 색에만 의존하지 않음 — 라벨 병기). 토글의 *현재 mode* 는 `aria-pressed` 로 명시.
- **모바일 대응**: 터치로 카메라 회전 / 핀치 줌, 컨트롤 패널은 하단 고정 (썸 영역). 모바일에선 시간 가속 최대치를 데스크톱보다 낮게 (배터리 보호). *모드 토글 버튼* 도 같은 영역.
- **브라우저**: WebGL2 지원 최신 Chrome / Safari / Firefox. WebGL2 미지원 시 fallback 메시지 (강제 폴리필 X)
