## 잘된 점

- **비례 압축 함수 도입 ([Light 6])** — 손튜닝 추측치를 *함수* 로 대체. `lib/scale.ts` 한 파일이 *"이 숫자 어디서 왔나"* 의 단일 답. sub-2-1 의 `data/planets.ts` 단일 소스 패턴이 *한 단계 더 추상화* 됨.
- **판별 합집합 (`RingData`)** — TypeScript 의 진짜 강점. `type: 'texture' | 'solid'` 한 줄로 *토성/천왕성의 본질적 차이* 가 타입 시스템에 박힘. `useTexture` hook 규칙 우회도 자연스럽게 해결.
- **group 3단 구조** — 공전 / 자전축 기울기 / 자전 의 좌표계 격리. [Light 4] 에서 의식적으로 분리해둔 결정이 [Light 7] axialTilt, [Light 8] Ring, [Light 9] 고리 회전 까지 *연쇄적으로 보상*.
- **데이터-코드 동기화 부채 의식적 해소** — sub-2-1 의 회고에 박힌 *axialTilt 부채* + 이번 [Light 4] 에서 발견한 *visualDistance 부채* + [Light 8] 의 *ring 부채* — 세 부채를 의식적으로 한 sub-phase 안에 정리.
- **TechSpec 의 *결정 뒤집기 두 번* (3차/4차)** — 갱신 헤더에 *왜 뒤집었나* 를 박아둠. 코드 변경뿐 아니라 *결정의 역사* 도 문서에 남음 — 헌장 §9 *Context Rot 방지* 의 실체.
- **PRD 의 영혼을 지킴** — 권이건의 *"실제 거리 기반 고정"* 제안이 PRD §3 의 *"지구가 사라지는 1초"* 영혼과 충돌. *답답함의 진짜 원인* 이 PRD 가 아니라 *내 첫 추정치* 임을 분리해서 진단 → 비례 압축으로 *영혼 유지 + 답답함 해소* 동시 달성.

## 막혔던 점 / 트러블슈팅

### 문제 1: 추측으로 코드 작성 — *Scene.tsx 추정본* 사건

**증상**: [Light 3] 에서 Scene.tsx 를 수정해야 하는데, 그 파일을 *받지 않은 상태로 추정본 생성*. *"실제 파일과 다르면 알려줘, 그대로 맞춰서 다시 줄게"* 라고 라벨링.

**원인**: 헌장 §11 의 *"모르는 건 모른다고 말하기 — 환각으로 메우지 않기"* 직접 위반. *"맞춰서 다시 줄게"* 라는 안전망이 *추측을 정당화* 하는 도구로 작용. sub-2-1 의 인수인계 React 18.3 오기재 사건과 *같은 종류 함정*.

**해결**: 권이건이 *"모르면 멈추고 나한테 물어보라고"* 알람. 즉시 추정 코드 폐기, 실제 파일 받은 후 작업 진행.

**포트폴리오 Troubleshooting 어필 포인트**:
- LLM/AI 협업의 *Anthropic 헌장 위반* 실시간 발견 + 즉시 정정
- *"틀리면 알려줘"* 의 안전망이 *추측을 정당화하는 함정* 이 됨을 인지
- 이 교훈을 *Claude 가 직접 회고 문서* 에 박아 다음 sub-phase 부터 *기존 파일 건드릴 일 = 무조건 파일 먼저 요청* 으로 정착

### 문제 2: [Light 4] 가 비어버린 사건

**증상**: 원래 Plan 의 [Light 4] = *"8개 행성 .map() 으로 렌더링"*. 그런데 실제 Scene.tsx 받아보니 sub-2-1 이 *선제적으로* `PLANETS.map((planet) => ...)` 깔아둠.

**원인**: sub-2-1 의 *"이미 .map() 패턴 채택 — 2-2 에서 8개 확장 시 Scene 코드 변경 0"* 약속이 실제로 작동. 그 사실을 [Light 2] data 확장 직후 *브라우저로 확인 안 함* → [Light 4] 시점에야 발견.

**해결**: Plan 갱신 — [Light 4] 자리에 *visualDistance + initialAngle* (행성이 안 보이던 진짜 부채) 가 들어오고, 원래 Light 들이 한 칸씩 밀려 9개 Light 로 확장.

**교훈**: *시각 컴포넌트 변경 후 빌드 통과 = 컴파일 OK*. *화면에 의도대로 보이는가* 는 *눈으로만 확인 가능*. [Light 2] 후 빌드만 보고 넘어간 게 *발견 지연의 원인*.

### 문제 3: 수성이 태양에 박힌 사건

**증상**: [Light 4] 후 화면 확인 — 수성이 *태양에 반쯤 들어가있음*. 권이건이 *"수성이 태양에 반쯤 들어가있는데?"* 발견.

**원인**: 내 첫 추정치 (`visualDistance = 8`, `visualRadius = 8`) 가 *수성 중심 = 태양 표면* 으로 정확히 충돌. 손튜닝 추측치를 박을 때 *최소 충돌 검증* 안 함.

**해결**: 두 단계:
1. 즉각: leva 슬라이더 (`distance`) 로 수동 조정 가능 안내
2. 근본: [Light 6] 비례 압축 함수 도입 — *손튜닝 자체를 제거*. 함수 한 줄이 8개 행성의 거리를 *수학적으로 일관* 하게 계산.

**교훈**: *손튜닝의 한계 = 추측치 박기*. 권이건의 *"각각 일일이 설정하는 거 너무 귀찮은데"* 한마디가 *비례 압축 함수 도입의 동기* 가 됨. 사용자의 *불편 보고* 가 *아키텍처 개선의 신호* — 이건 SkyWear 시기에도 통하던 패턴.

### 문제 4: PRD 영혼 흔들림 — *"실제 거리 기반 고정"* 제안

**증상**: 수성 박힘 발견 후 권이건이 *"실제 거리를 기반으로 해서 고정값으로 하면 좋을 거 같은데"* 제안. 이는 PRD §3 의 *"거리 스케일 토글 = 지구가 사라지는 1초"* 영혼과 정면 충돌.

**원인**: 두 가지가 섞임 —
1. *답답함의 즉각적 불편* (수성 박힘, 행성 안 보임)
2. *시각화 철학의 근본 결정* (시각/실제 토글 vs 비례 고정)

이 둘이 같은 발언 안에 섞여 *PRD 재논의* 처럼 보임.

**해결**: 두 결정 분리:
- *답답함의 진짜 원인* = 내 첫 추정치. [Light 6] 비례 압축 도입으로 해결
- PRD 의 *1초의 충격* 메시지는 *영혼* 이라 유지. 작성한 지 이틀밖에 안 됨 — *Context Rot 의 가장 흔한 패턴* (이틀 만에 결정이 흐려짐)

**교훈**: *권이건의 사용자 시점 피드백* 과 *PRD 의 작성자 시점 결정* 을 분리해서 들어야 함. 즉각 불편이 *항상 PRD 재논의의 정당한 이유는 아님*. 그러나 *비례 압축 도입은 영혼 유지 + 답답함 해소의 양수겸장* — 이런 우아한 절충이 가능할 때 받아들임.

### 문제 5: *변경 파일 다 보여줘* 알람 — 패턴 스니펫 던지기

**증상**: [Light 3] 에서 Sun.tsx 와 sun.ts 새 파일만 *전체 코드* 로 주고, Scene.tsx 변경은 *패턴 스니펫* 으로 *"여기 추가하면 돼"* 식 처리.

**원인**: 헌장 §10 *"속도 vs 이해 → 이해 우선"* 위반. *복붙으로 끝* 인 형식이 아니라 *권이건이 코드를 직접 끼워넣는* 부담을 떠넘김.

**해결**: 권이건이 *"새 파일이랑 변동된 파일들 다 보여줘야지.. 뭐하는거야?"* 알람. 이후 *기존 파일 건드릴 일 = 전체 파일 출력* 패턴 정착.

**교훈**: *작업 결과의 응집도* 가 중요. 새 파일 / 수정 파일 / 영향 파일을 *한 답변에 다 모아서* 보여주는 게 헌장의 *"PCTC Light 한 사이클 = 동작하는 가장 작은 단위"* 정신.

### 문제 6: 답변 길이 한도 차서 중간 끊김 — *Light 6 사건*

**증상**: [Light 6] 비례 압축 도입 작업이 *TechSpec 갱신 + 7개 파일 변경* 의 큰 단위. 한 답변에 다 담으려다 *짚어둘 포인트 / T / C / commit 메시지* 가 빠진 채로 끊김.

**원인**: Claude 가 *작업 단위 응집도* 와 *답변 길이 한도* 의 균형을 못 잡음. 하나의 *논리적 단위* 를 *물리적 한도 안에* 못 담음.

**해결**: 권이건의 *"한도가 다 차서 중간에 멈췄네.. 빼먹은 거 다 보여주고"* 알람. 다음 답변에서 보충.

**교훈**: 큰 변경 작업은 *답변을 의식적으로 두 차례로 분리* (예: *코드 파일 → 다음 답변에서 검증/회고*). 헌장 §9 *Context Anxiety 방지* 의 변형 — *Claude 가 한 답변에 다 담으려는 욕심* 도 anxiety.

## 🌌 이번에 알게 된 우주의 한 조각

**천왕성 고리 발견 — 1977년 3월 10일의 우연**

천문학자들이 SAO 158687 이라는 별이 *천왕성 뒤로 잠시 가려지는 현상* (occultation) 을 관측하려고 카이퍼 공중 천문대 (KAO, 비행기에 망원경 탑재) 가 별빛을 추적 중이었어. *천왕성 대기의 정보를 얻기 위한* 일상적 관측.

그런데 — 천왕성이 별을 가리기 *35분 전*, 별빛이 갑자기 깜빡임. 9번. 그리고 천왕성이 지나간 후 *대칭으로 또 깜빡임*. 9번.

순간 천문학자들의 머리에 떠오른 답: *별 앞을 천왕성 외에 9개의 무언가가 더 지나갔다*. 천왕성 둘레의 **9개 (당시 발견 수) 좁고 어두운 고리**. 토성 이후 200년 만의 두 번째 고리 행성 발견.

천체관측의 우아함이 여기에 있어 — *직접 본 게 아니라 별빛이 깜빡인 패턴* 으로 *존재하지 않는 것처럼 보였던 것* 을 발견. 어두운 고리는 *너무 어두워서 망원경으로 직접 못 봐*. 별 가림이라는 우연이 *어둠 속의 실루엣* 을 드러냈어.

**그런데 — 이 발견 방법이 Phase 6 의 외계행성 사냥과 정확히 같은 원리야**. *별빛의 미세한 깜빡임* 으로 별 앞을 지나가는 행성의 존재를 추론하는 게 **transit method**. 1977년 천왕성 고리 발견이 *별 가림 + 패턴 인식* 의 첫 적용 사례. 코드 `<Ring />` 한 줄에 *Phase 6 BLS 알고리즘의 조상* 이 박혀있어.

**부수적으로 배운 것**:

- **황도면 (xz 평면) 의 사연** — 45억 년 전 회전하는 가스 구름이 자체 중력으로 *원반 모양으로 압축* 된 결과. 우리 은하도 같은 모양 → *밤하늘 은하수가 띠처럼 보이는* 이유. 그리고 transit method 가 가능한 *물리적 토대* (외계행성계도 다 원반).
- **베버-페히너 법칙** — *우리 뇌가 세상을 보는 방식* = 제곱근/로그 압축. 코드 `(real / EARTH_RADIUS) ^ 0.5` 의 *0.5* 는 *우리 인지가 이미 그렇게 보고 있던 방식* 을 코드로 재서술. 별 등급 / 데시벨 / 리히터 모두 같은 원리.
- **태양은 *흰 별*** — 우주에서 보면 완벽한 흰색. *노란 태양* 은 *지구 대기 산란* 의 결과. `lightColor: '#fff5e1'` 은 *대기 효과를 약간 흉내낸 절충*.
- **금성의 *이중 표현*** — `rotationPeriod_hours: -5832.5` (역회전) + `axialTilt_deg: 177.4` (거의 뒤집힘) 은 *같은 사실의 두 표현*. NASA 규약 그대로.
- **태양계 질량 분포** — 태양이 99.86%, 나머지 0.14%. 행성 + 위성 + 소행성 + 혜성 *모두 합쳐도* 태양의 0.14%.
- **토성 고리 두께 평균 10m** — 폭은 수십만 km 인데 두께는 *건물 한 층*. 거의 *수학적 평면*.
- ***교육적 거짓말* 의 다중 층** — Solar Textures 의 텍스처도 *합성본* / 금성은 *레이더 데이터* 시각화. 우리는 *시각화된 데이터를 또 시각화*. 두 거짓말 다 *진실을 향한 거짓말*.
- **45억 년 전 두 충돌의 흔적이 코드 두 줄에** — 지구 자전축 23.5° (테이아 충돌 → 달 형성), 천왕성 자전축 97.77° (지구 크기 천체 충돌). *우리가 여기 있는 이유* 와 *천왕성의 42년 백야* 가 같은 종류의 *우주적 우연*.
