## 1. 한 문장 요약

**방문자는 태양계를 인터랙티브하게 탐험하면서, 시각화와 실제 사이의 거리를 깨닫는다.**

---

## 2. 동기 (Why)

어릴 적, 칼 세이건의 *Pale Blue Dot* 사진을 처음 봤을 때 — 보이저 1호가 60억 km 밖에서 찍은 지구가 단 한 픽셀의 푸른 점이었던 그 사진. 그 이미지는 *우주에서 우리가 얼마나 작은가* 를 가르치기보다, *우주가 얼마나 큰가* 를 가르쳤다. Phase 2는 그 깨달음을 코드로 다시 쓰는 일이다.

우리가 책에서, 다큐에서, 심지어 NASA 공식 일러스트에서 보는 태양계는 모두 *거짓말*이다 — 정확하게는, *교육적 거짓말*. 행성들이 그렇게 가까이 모여 있다면 우리는 안드로메다보다 빠른 충돌을 매일 봤을 것이다. 진짜 태양계는 *대부분 빈 공간*이다. Phase 2는 그 빈 공간을 보여주는 Phase다.

기술적으로는 — Three.js + React Three Fiber로 첫 본격 3D 작업, 디자인 토큰 시스템(Phase 1)이 셰이더까지 흘러가는지 검증, 그리고 *"수치를 그대로 쓰면 보이지 않는다"* 라는 시각화의 본질적 트레이드오프를 의식적으로 다루는 첫 무대.

---

## 3. 사용자 시나리오

**60초 시나리오**:

1. **(0~10s)** 페이지 로드. 어두운 우주, 중앙에 태양. 8개의 행성이 자기 궤도를 도는 *시각적 모드*가 기본. 한눈에 *"아, 태양계"* 가 들어온다.
2. **(10~25s)** 행성에 호버하면 이름 + 기본 정보. 클릭하면 카메라가 그 행성으로 줌인. 지구를 클릭하면 달까지 같이 보인다.
3. **(25~40s)** 화면 하단 컨트롤 패널: **[거리 스케일 토글]** **[시간 속도 슬라이더]** **[정지 ⏸ / 리셋 ↺]** **[카메라 리셋]**. 거리 스케일을 *"실제 비율"* 로 옮기는 순간 — 행성들이 화면 밖으로 흩어지고, 지구가 점이 되어 사라진다.
4. **(40~60s)** *"어..."* 의 순간. 시간 정지로 멈추고 둘러보거나, 시간 속도를 최대로 올려 해왕성이 한 바퀴 도는 데 *얼마나 오래*인지 체감한다. 길을 잃으면 카메라 리셋 한 번이면 처음 시점으로.

**기억에 남길 한 순간**: *거리 스케일 토글을 누르는 순간 지구가 사라지는 1초*. 이 1초를 위해 나머지 모든 게 존재한다.

## 4. 범위

### ✅ 포함 (6개)

1. **8개 행성 + 태양 3D 시각화** — 저폴리곤 sphere + NASA 텍스처
2. **달** — 지구의 위성으로 함께 표시 (*"우리 위성"* — 거리 토글 시 또 하나의 깨달음 포인트)
3. **거리 스케일 토글** — 시각적 ↔ 실제 비율
4. **시간 컨트롤** — 슬라이더 (정지 / 1x / 100x / 10000x / 100000x) + 정지 버튼 + 리셋 버튼
5. **행성 호버/클릭 인터랙션** — 호버 시 정보, 클릭 시 카메라 줌인 + 카메라 리셋 버튼
6. **단순 starfield 배경** — 별자리 라인 없음 (perspective 문제)

### ❌ 제외

- **별자리 라인** → perspective 문제 (지구 시점에 묶이는 데 카메라가 자유로움)
- **달 외 위성들** (목성 갈릴레이 위성 등) → 추후 Phase 또는 미니 프로젝트
- **소행성대 / 카이퍼 벨트** → 시각적 노이즈, 메시지 흐림
- **행성 표면 상세** (분화구, 폭풍 등) → 텍스처로 충분, 절차적 디테일은 과잉
- **외계행성** → Phase 6
- **중력 시뮬레이션** (행성이 서로 끌어당기는) → Phase 3 N-body

## 5. 성공 기준

### 정성적
- *"거리 스케일 토글 순간"* 이 실제로 충격을 준다 (자체 테스트: 처음 본 친구 3명이 *"어..."* 라고 말하는가)
- 시각화와 실제의 거리감이 *수치*가 아니라 *체감*으로 전달된다
- 모바일에서 만져봐도 답답하지 않다
- 실수해도 괜찮은 컨트롤 — 시간 정지, 카메라 리셋이 안전망 역할

### 정량적
- 데스크톱 **60fps 유지** (8개 행성 + 달 + starfield + 카메라 컨트롤)
- 모바일 **30fps 이상**
- **Lighthouse Performance 85+** (3D는 90+ 어려움)
- **첫 인터랙티브까지 < 3s** (3D 자산 로드 포함)

---

## 6. 비기능 요구사항

- **성능**: WebGL2 기반, 60fps 데스크톱 / 30fps 모바일. 지오메트리 최소화 — 행성 = 저폴리곤 sphere + 텍스처
- **접근성**: 토글/컨트롤은 키보드로도 조작 가능, 색맹 대비 (행성 식별을 색에만 의존하지 않음 — 라벨 병기)
- **모바일 대응**: 터치로 카메라 회전 / 핀치 줌, 컨트롤 패널은 하단 고정 (썸 영역). 모바일에선 시간 가속 최대치를 데스크톱보다 낮게 (배터리 보호)
- **브라우저**: WebGL2 지원 최신 Chrome / Safari / Firefox. WebGL2 미지원 시 fallback 메시지 (강제 폴리필 X)
